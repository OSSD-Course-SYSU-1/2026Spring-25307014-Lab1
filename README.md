# Open Source Practice Repository

## 我打算如何参与开源（贡献想法）
- 从小任务开始：修复文档错别字/格式问题、补充使用说明与示例。
- 复现并提交 issue：提供复现步骤、环境信息、日志与截图。
- 提交小型 PR：优先做低风险改动（README、注释、测试用例、CI 配置）。
- 参与讨论：在 issue/讨论区给出可执行建议与验证结果。

## 相比上次：我的想法有什么变化？
相比上一次，我从“想参与但不知道做什么”变成了更具体的计划：先从文档与可复现 issue 这类低风险贡献入手，逐步熟悉 Git/GitHub 的完整协作流程（issue → PR → review）。

## 本学期课程学习目标
- 熟悉 Git/GitHub 基本流程：clone、branch、commit、push、PR、code review。
- 学会用 issue 驱动协作：提问、定位、跟进、关闭问题。
- 掌握基础项目规范：README、CHANGELOG、版本号、许可证、贡献指南。
- 形成可复用的开发习惯：清晰提交信息、最小可用变更、可复现的 bug 报告。
